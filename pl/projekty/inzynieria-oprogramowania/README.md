# Delivery Schedule Management System – Design

A project from the **Software Engineering** course (2025).

This is not an implementation of a system, but a complete design process – from vision and requirements analysis, through use cases and object model, to dynamic diagrams.

## System description

The system is designed to improve delivery schedule management for a fictional logistics company **Phoenix Sp. z o.o.** The company operates in the Pomeranian region, serves ~80 clients per month, owns 20 vehicles and employs 15 drivers.

## Design stages

| Stage | Name | Description |
|-------|------|-------------|
| 1 | System Vision | Organization analysis, stakeholders, functional and quality requirements (MoSCoW), constraints |
| 2 | Use Cases | Use case diagram, 20 structured descriptions with preconditions, scenarios and exceptions |
| 3 | Object Model | Class diagram with attributes, operations, associations, generalization and composition |
| 4 | Dynamic Diagrams | 3× sequence, 1× communication, 1× activity, 1× state |
